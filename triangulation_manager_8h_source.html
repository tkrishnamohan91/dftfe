27 /**
28 * @brief This class generates and stores adaptive finite element meshes for the real-space dft problem.
29 *
30 * The class uses an adpative mesh generation strategy to generate finite
31 * element mesh for given domain based on five input parameters: BASE MESH
32 * SIZE, ATOM BALL RADIUS, MESH SIZE ATOM BALL, MESH SIZE NEAR ATOM and MAX
33 * REFINEMENT STEPS (Refer to utils/dftParameters.cc for their corresponding
34 * internal variable names). Additionaly, this class also applies periodicity
35 * to mesh. The class stores two types of meshes: moved and unmoved. They are
36 * essentially the same meshes, except that we move the nodes of the moved
37 * mesh (in the meshMovement class) such that the atoms lie on the nodes.
38 * However, once the mesh is moved, dealii has issues using that mesh for
39 * further refinement, which is why we also carry an unmoved triangulation.
40 * There are other places where we require an unmoved triangulation, for
41 * example in projection of solution fields from the previous ground state in
42 * stucture optimization.
43 *
44 * @author Phani Motamarri, Sambit Das, Krishnendu Ghosh
45 */
